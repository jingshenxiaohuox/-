# 1. 单播、组播、广播的区别？
## 单播 (Unicast)：一对一通信。就像两个人打电话，数据包只发给指定的单个目标 IP（比如你项目中建立的 TCP 连接）。

## 广播 (Broadcast)：一对所有通信。像村里的大喇叭，把数据发给同一子网内的所有设备（比如局域网里的 ARP 寻址）。容易引发广播风暴。

## 组播 (Multicast)：一对一组通信。只发送给加入了特定“组播组”的设备。结合你的网关项目可以提一句：“就像视频流分发，多端监控不需要建立多次单播，用组播可以极大节省服务器网络带宽。”

# 2. 原码、反码、补码
## 原码：最高位是符号位（0正1负），剩下的是真值。缺点是 0 有两种表示（+0和-0），且加减法运算复杂。

## 反码：正数和原码一样；负数是符号位不变，其他位按位取反。

## 补码（重点）：正数和原码一样；负数是反码 +1。

## 面试官想听的：计算机底层全部采用补码来存储和运算。它解决了 0 的唯一性问题，并且把减法完美转化为了加法，让 CPU 的 ALU（算术逻辑单元）设计大大简化。

# 3. struct和类的区别
## 默认访问权限：C++ 中 struct 默认的成员是 public 的，而 class 默认是 private 的。

## 默认继承权限：struct 默认是 public 继承，class 默认是 private 继承。

## 补充：在 C 语言里 struct 只能存数据不能有函数；但在 C++ 里，struct 也可以有构造函数、多态等，本质上和 class 已经非常接近了，常用来定义纯数据结构（比如你写的自定义协议包头）。

# 4. 引用和指针的区别
## 本质：指针是一个变量，里面存的是地址，会占用内存空间（4或8字节）；引用只是一个别名，不分配额外内存。

## 初始化：指针可以不初始化，也可以指向 nullptr；引用必须在定义时初始化，且绝对不能为 null。

## 可变性：指针可以随时改变指向其他内存；引用一旦绑定了一个对象，就永远不能更改绑定。

# 5. #include <> 和 "" 的区别
## <>：告诉编译器直接去系统标准库的路径下找这个头文件（例如 <iostream>）。

## ""：告诉编译器先在当前源文件所在的目录下找；如果找不到，再去系统标准库路径找（例如 "protocol.h"）。

# 7. 重载、覆盖（重写）的区别
## (面试官常挖坑，注意区分这俩)

## 重载 (Overload)：在同作用域内（比如同一个类），函数名相同，但参数列表不同（类型或个数）。返回值不同不算重载。

## 重写/覆盖 (Override)：发生在父子类之间。子类重新定义了基类的 virtual（虚）函数。函数名、参数列表、返回值必须完全一致。这是实现多态的核心。

# 8. Go和C++的区别
## 内存管理：C++ 需要程序员手动管理内存（new/delete 或 智能指针），容易内存泄漏；Go 自带垃圾回收器（GC），自动管理内存。

## 并发模型（核心）：C++ 并发通常依靠多线程（std::thread）或底层的 epoll 机制，心智负担重；Go 的强项在于原生支持轻量级协程（Goroutine）和通道（Channel），写高并发网络服务极其简单。

## 生态与编译：C++ 语法复杂历史包袱重，但底层性能极致；Go 编译速度极快，工程化好，但在处理极低延迟场景时会受 GC 暂停的影响。

# 9. 线程间同步方式
## 互斥锁 (Mutex)：最基础的加锁，保证同一时刻只有一个线程能进入临界区操作共享数据。

## 条件变量 (Condition Variable)：配合互斥锁用，用于线程间的等待和通知机制（比如经典的生产者-消费者模型，队列空了消费者就挂起等待）。

## 读写锁 (Read-Write Lock)：允许多个读者同时读，但写者独占。适合“读多写少”的场景。

## 信号量 (Semaphore)：控制同时访问某个特定资源的线程数量上限。

# 11. 介绍两种单例模式
## 懒汉式：不到万不得已不创建。首次调用 getInstance() 时才 new 出实例。注意：在多线程下不安全，需要加锁（双重检查锁定 DCL），或者直接利用 C++11 的特性使用局部静态变量（Magic Static，最推荐）。

## 饿汉式：不管用不用，程序一启动（类加载时）就立刻创建实例。天生线程安全，但可能会拖慢程序启动速度或造成内存浪费。
